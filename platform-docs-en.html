Section 1

Core / Homepage

The entry points of the platform — where visitors discover listings, search by category, and orient themselves to how BulanVarMı? works.

🏠
Page 01

Home Page

/

What this page is

The home page is the first thing any visitor sees. It gives an immediate snapshot of the platform: how many items have been reported lost, how many found, and how many cases resolved. It serves as both a landing page and a quick navigation hub.

What users can do here

  • See live statistics — total listings, lost count, found count, resolved count — that count up with an animated counter on load.
  • Browse the category shortcut grid: Phone, Wallet, Keys, Bag, Laptop, Pet, Other, and an "Urgent" highlight. Each tile shows the count of active listings in that category and links directly to a filtered search.
  • Enable Nearby Listings: the page asks for browser location permission and then shows up to 8 active listings within 20 km of the user on a horizontal scroll strip.
  • See a Recently Viewed strip (stored in browser local storage) showing items the user has opened in the current session.
  • Navigate to key sections via footer links: Privacy Policy, Terms of Use, Success Stories, Lost Property Offices, Favourites, All Listings.

Important behaviour

Statistics and categories are visible to everyone. However, clicking a listing card requires being logged in — unauthenticated visitors are redirected to login, then back to where they were trying to go.

Who uses it
  • All visitors, logged in or not
  • Returning users checking their recent items
  • People who just discovered the platform
Key features
  • Animated stat counters
  • Category quick-filter grid
  • GPS-based nearby listings
  • Recently viewed (local storage)
Login required?

No for browsing stats/categories. Yes for opening a listing.

Page 02

Search & Browse Listings

/search

What this page is

The main discovery page. It shows all active, approved listings in a card grid and gives users a full toolkit to narrow results until they find what they are looking for — or confirm a match hasn't been posted yet.

Filtering and sorting options

  • Keyword search: Searches across title, description, and location. Updates 400 ms after the user stops typing (debounced).
  • Type tabs: All, Lost, Found — filter to see only one type.
  • Category dropdown: Phone, Wallet, Key, Bag, Laptop, Watch/Jewellery, ID/Documents, Pet, Other.
  • Sort order: Newest first, Oldest first, Most viewed.
  • Date range: Narrow to listings created within a specific window.
  • Location + radius filter: Pick a point on the map and set a radius; only listings with GPS coordinates within that distance are shown. A mini-map appears in the results when this filter is active.
  • Hide Resolved toggle: Remove already-closed listings from view.

Additional features

  • Search history: Last 5 searches stored in the browser, shown as quick-tap chips under the search box.
  • Save Alert button: Saves the current search as an alert — the user receives an email when a new listing matches those criteria.
  • Infinite scroll: Results load 12 at a time; scrolling to the bottom loads the next batch automatically.
  • Full-screen map: "Open Map" button shows all filtered results on the full-screen map page.
Who uses it
  • Users searching for their lost item
  • People who found something checking if it was reported
Each card shows
  • Thumbnail photo
  • Lost / Found / Resolved badge
  • "New" badge (under 24 hours)
  • Category, location, time
  • View count
Login required?

Yes — unauthenticated visitors are redirected to login.

Page 03

Map View

/map

What this page is

A full-screen interactive map that plots all active, approved listings that have GPS coordinates. This gives users a visual, geographic way to find items near a specific location rather than reading through a list.

What users can do here

  • See coloured map markers: red for lost listings, green for found listings.
  • Click any marker to see a popup with the item's title, category, and a link to its full detail page.
  • Filter by type (All / Lost / Found) using a tab bar at the top.
  • Filter by category using a collapsible category panel.
  • Enable a radius filter: the page requests GPS location and then shows only listings within a chosen radius (1, 5, 10, 25, or 50 km).
  • Up to 500 listings are loaded at once (newest first). Only listings with both latitude and longitude appear.

How it fits in

The map is particularly useful when a user knows roughly where an item was lost or found and wants to check visually. It complements the text search on /search.

Who uses it
  • Users who prefer a visual search
  • People searching in a specific neighbourhood
Colour key
  • 🔴 Red = Lost listing
  • 🟢 Green = Found listing
Login required?

Yes — behind an auth guard.

Page 04

Report a Lost Item

/lost/report

What this page is

A 3-step wizard where a user creates a "Lost" listing for something they have lost. The form collects enough detail so the AI matching engine can automatically find a corresponding "Found" listing.

Step 1 — Basic Information

  • Item name (up to 80 characters) — e.g., "Black iPhone 15 Pro".
  • Category — chosen from a grid: Phone, Wallet, Key, Bag, Laptop, Watch/Jewellery, ID/Documents, Pet, Other.
  • Description (up to 1000 characters) — appearance, special marks, circumstances of loss.
  • AI Description Suggestion: once a title and category are filled in, one click asks the AI to draft a suggested description. The user can edit it freely before continuing.
  • If "Pet" is chosen, additional fields appear: species, breed, colour/pattern, age/size, microchip number.

Step 2 — Location & Date

  • Location text field — where the item was lost (e.g., "Kadıköy / Istanbul").
  • Auto-detect button: uses the browser's GPS to fill the location field automatically via reverse geocoding.
  • Map pin button: opens a map modal where the user can drop a precise pin.
  • Date and optional time of loss.

Step 3 — Photos

  • Upload 1–5 photos. Each image goes through a crop/resize tool before upload.
  • The first uploaded photo triggers AI category detection: the AI analyses the image and suggests a category and title if those fields are still empty.

On submission

The listing is saved with status "pending" and awaits admin approval before going public. A duplicate-detection warning appears if the user already posted in the same category in the last 24 hours. A draft is auto-saved to localStorage so work isn't lost if the user navigates away.

Who uses it

Any logged-in user who lost an item and wants help finding it.

AI features
  • ✨ AI description writer
  • ✨ AI image category detection
After submitting

User is redirected to My Listings and earns +10 points. The listing goes live after admin approval.

Page 05

Report a Found Item

/found/report

What this page is

Identical in structure to the Lost Report page, but for users who found an item belonging to someone else and want to help it get back to its owner. The language shifts to "Where did you find it?" and "What did it look like when you found it?"

Key differences from the Lost Report

  • The listing is saved with type = "found".
  • Step 2 asks "Where and when did you find it?" instead of "Where did you lose it?"
  • The description prompt encourages the finder to describe what was inside or near the item, helping the true owner recognise it.
  • All other steps and AI features — AI description, image detection, GPS, map pin, crop tool, draft saving, duplicate check — are identical to the Lost Report.

What happens after submission

Same as the Lost Report: held for admin moderation. Once approved, the AI matching engine scans it against existing Lost listings and notifies both parties if a strong match is found. The finder earns +10 points.

Who uses it

Logged-in users who found something and want to connect it with its owner.

Same features as Lost Report
  • 3-step wizard
  • Pet-specific fields
  • AI description & image AI
  • GPS auto-detect & map pin
  • Draft auto-save
  • Duplicate detection

Page 06

Item Detail Page

/items/[id]

What this page is

The full detail view of a single listing. This is the most feature-rich page on the platform — it is where the core interaction between the person who lost an item and the person who found it actually takes place.

What a visitor sees

  • A large photo gallery (up to 5 photos) with a thumbnail strip to navigate between them.
  • Item title, type badge (Lost / Found), category badge, and status (Resolved if closed).
  • Location and date/time details.
  • Total view count and number of "I saw it" sighting reports.
  • Full description text.
  • The listing owner's truncated email and their average star rating from past transactions.
  • Up to 3 AI-matched listings from the opposite type, each showing a percentage similarity score.
  • A grid of similar listings in the same category.

Actions for non-owners (logged in)

  • On a Found listing: "This item is mine!" — opens a claim modal to prove ownership (name, where lost, brand/model, distinctive feature).
  • On a Lost listing: "I saw this!" — sighting modal to report where they last saw it. Also "I found it!" — same claim modal.
  • Add/remove from Favourites.
  • Share: native share, copy URL, WhatsApp, Twitter/X, QR code, poster, social-image generator.
  • Report the listing as spam, misleading, inappropriate, duplicate, or other.
  • If a match score is ≥ 85%, a "Send Message" button appears directly on the match card.
  • Rate the listing owner (stars + optional comment) once the listing is resolved.

Actions for the listing owner

  • Three-dot menu: Edit listing, Renew for 30 more days, Mark as Resolved, Delete.
  • When resolved, the owner is prompted to rate the finder/claimant.
  • On mobile, a sticky bottom action bar shows the primary action button.
Who uses it
  • Owner — to manage their listing
  • Claimants — to prove ownership
  • Finders — to report a sighting
  • Any logged-in user browsing
AI on this page

AI-matched listings load in the background and are displayed with a percentage similarity score. A score ≥ 85% enables direct messaging.

Pending listings

A listing awaiting admin approval is only visible to its creator. Everyone else sees a "Not Found" screen.

Page 07

Edit Listing

/items/[id]/edit

What this page is

A form that lets the listing owner update any detail of their published listing. Only the owner can access it — anyone else sees a "You cannot edit this listing" message.

What can be changed

  • Title, category, location (text field and map pin), date, and description.
  • Replace the main photo — the user picks a new image, which goes through the same crop tool used at listing creation.

After saving

Changes are saved to the database and the AI matching index for this listing is regenerated so the matching engine reflects the updated content. The user is redirected back to the item detail page.

🙋
Page 08

Ownership Claim Page

/items/[id]/claim

What this page is

A dedicated form where a logged-in user formally claims that a "Found" listing belongs to them. This is an alternative to the claim modal on the Item Detail page — both do the same thing, but this page gives more space for typing a detailed response.

What the user fills in

  • Full name (required) — how the claimant wants to be identified.
  • When they lost it — date (required).
  • Where they lost it — location text (required).
  • Brand / model — optional, e.g., "Apple AirPods Pro".
  • Distinctive feature (required) — a detail only the true owner would know, e.g., "scratch on the bottom-right corner".
  • Additional note — any other context (optional).

What happens after submission

A claim record is created with status "pending". The listing owner receives an in-app notification and an email. The owner can approve or reject the claim in their profile's "Incoming Claims" tab. If approved, a private conversation channel is automatically opened between both users.

Page 09

Printable Poster

/items/[id]/poster

What this page is

A clean, print-optimised A4 layout of the listing that the owner can print and put up around the neighbourhood where the item was lost or found. All navigation is stripped away and the dark UI is replaced with a bright white layout for paper printing.

What the poster contains

  • A large "LOST" or "FOUND" header in bold colour (amber for lost, green for found).
  • The item photo, title, description, category, location, and date.
  • A call-to-action with the listing URL so anyone who spots it can visit and submit a claim or sighting.
  • BulanVarMı? branding and website address.

How to use

A top control bar (hidden when printing) has a "Print / Save as PDF" button which triggers the browser's native print dialog. The user can then print physically or save as PDF.

Page 10

Messages List

/messages

What this page is

An inbox listing all conversations the logged-in user is part of. Conversations are created automatically when a listing owner approves an ownership claim — both the owner and the claimant are added. Conversations cannot be started manually; they are always tied to a specific listing.

What users can do here

  • See all conversations sorted by creation date (newest first).
  • Each entry shows the item title, the other participant's email, creation time, and an unread count badge if there are new messages.
  • Click a conversation to open the full chat.
  • Delete a conversation (with confirmation). Deletion removes all messages permanently.
  • New conversations and incoming messages update in real time — no page refresh needed.
Who uses it

Any user who has an approved claim, either as the item owner or as the claimant.

Real-time

Unread counts and new conversations appear live via Supabase realtime — no refresh needed.

Login required?

Yes — unauthenticated users see a "Please log in" panel.

Page 11

Conversation Detail

/messages/[conversationId]

What this page is

The actual chat window for a single conversation. It looks like a standard messaging app: the user's own messages appear on the right in blue, the other person's messages appear on the left in dark grey. Only the two participants can access the conversation.

What users can do here

  • Send text messages — messages appear immediately (optimistic UI) and are confirmed by the server in the background.
  • Send photos — uploaded to storage and displayed inline in the chat.
  • Use message templates (Quick Replies) — a panel with pre-built phrases to save typing. Users can also add their own custom templates.
  • Delete their own messages (with confirmation).
  • See read receipts: ✓ = sent, ✓✓ = the other person has read it.
  • See a live connection status badge (Connecting / Live connection active / Connection error).
  • System messages appear in a distinct green bubble — e.g., "The listing has been resolved."

Real-time behaviour

The page subscribes to a Supabase realtime channel. Incoming messages from the other user appear instantly. When the chat is open, incoming messages are automatically marked as read.

Who uses it

The listing owner and the approved claimant — only these two people can see this conversation.

Features
  • Text + image messaging
  • Read receipts (✓✓)
  • Quick reply templates
  • System message bubbles
  • Live realtime updates
  • Message delete

Page 12

Notifications

/notifications

What this page is

An in-app notification centre that logs all platform events relevant to the logged-in user, in reverse chronological order.

Types of notifications shown

  • A new ownership claim was submitted on one of the user's listings.
  • A claim the user submitted was approved or rejected.
  • A new message arrived in one of the user's conversations.
  • The AI engine found a potential match for one of the user's listings.

Behaviour

When the page loads, all unread notifications (shown with a blue highlight) are automatically marked as read. Each notification links directly to the related listing or conversation. The page shows the last 50 notifications.

Page 13

My Profile

/profile

What this page is

The personal account hub. It combines everything about the logged-in user in one place: their listings, claims, favourites, notification preferences, statistics, and account security settings. The page is divided into tabs.

Tab — My Listings

Shows all listings created by the user (active, resolved, pending, expired) as cards, each linking to the listing's detail page.

Tab — Incoming Claims

Lists all ownership claims submitted by other users on the current user's "Found" listings. Each claim shows the claimant's name, email, where they say they lost the item, and their stated distinctive feature. The owner can Approve (which opens a conversation) or Reject each claim.

Tab — Sent Claims

Shows all claims the user has submitted on other people's listings, with the current status (Pending / Approved / Rejected).

Tab — Notifications

Toggle switches for email notifications (claims, messages, AI matches, weekly digest), Privacy Mode (hides email from other users), and browser push notification subscription.

Tab — Statistics

A table showing each listing with its view count, favourite count, and claim count. Summary cards at the top show totals.

Tab — Account

Phone number verification (SMS OTP), SMS notification toggle, referral code generation and sharing, points and badge display, and two-factor authentication (TOTP via Google Authenticator).

Profile header

Shows avatar, full name, email, and email-verification status. Buttons: Upload Photo, Edit Profile (name), Log Out.

Who uses it

The logged-in user managing their own account.

Tabs
  • My Listings
  • Incoming Claims
  • Sent Claims
  • Notifications / Privacy
  • Statistics
  • Account (2FA, phone, referral)
Points earned by action
  • +10 — creating a listing
  • +50 — resolving a listing
  • +20 — approved claim
  • +5 — sighting report

Page 16

Search Alerts

/alerts

What this page is

A page where users set up saved searches that automatically send an email notification whenever a new listing matching their criteria is posted. Ideal for people who lost something and want to be notified automatically instead of checking the site daily.

Creating an alert

  • A keyword (searches listing title and description).
  • Listing type: All / Lost / Found.
  • Category filter.
  • Location name (text).
  • Optional GPS radius — the page requests browser location; alerts fire only for listings within the chosen distance.

Managing existing alerts

  • Each alert is shown as a row with its filter tags (keyword, category, location) and creation date.
  • Toggle on/off to pause an alert without deleting it.
  • Delete permanently.
  • Maximum 10 alerts per user. A usage counter is shown (e.g., "3/10 alerts in use").
  • Shows the date of the last notification sent for each alert.
Who uses it

Users who lost something and want passive notifications rather than checking the site daily.

How alerts fire

A scheduled cron job checks all active alerts against newly added listings and sends email notifications to matching users.

Page 19

Login

/auth/login

What this page is

The sign-in page. It offers three different ways to log in, catering to different user preferences.

Login methods

  • Google OAuth: "Sign in with Google" button — opens Google's authentication and returns the user to where they were trying to go. The redirect URL is preserved throughout the OAuth flow.
  • Email + password: Standard form with email address and password. A "Forgot my password" link is provided.
  • Phone (SMS OTP): User enters their phone number with country code; a 6-digit SMS code is sent. After entering the code, they are logged in. If the phone number has no email attached, the user is redirected to the Complete Profile page first.

After login

The user is sent to the page they were originally trying to reach (captured in the URL's ?redirect= parameter), or to the home page if no redirect was set.

Login methods
  • 🔵 Google (OAuth)
  • 📧 Email + password
  • 📱 Phone (SMS OTP)
Smart redirect

If a user was sent to login from a protected page, they return to that page after logging in — not to the home page.

📝
Page 20

Register

/auth/register

What this page is

The account creation page. A 2-step wizard that first asks what type of account the user wants, then collects their credentials.

Step 1 — Account type

  • Personal Account: for individuals who want to post lost/found listings.
  • Business / Organisation Account: for companies and institutions. Selecting this reveals additional fields (company name, business type) in step 2.

Step 2 — Registration method

  • Email tab: Full name, email address, password (minimum 6 characters). Business accounts also require company name and business type.
  • Phone tab: Phone number → SMS OTP → verify. If the phone number is new, the user proceeds to Complete Profile to add an email.
  • Referral code: If the URL contains a ?ref= parameter (from a friend's referral link), it is pre-filled and the referral is recorded.

After registration (email method)

The user is redirected to the Email Verification page and receives a confirmation email. The account is active only after clicking the link in that email. The system checks the email against a blacklist before creating the account.

Account types
  • 👤 Personal
  • 🏢 Business/Organisation
Registration methods
  • 📧 Email + password
  • 📱 Phone (SMS OTP)
Referral

If registered via a referral link (?ref=CODE), the referral is tracked and both users may earn benefits.

Forgot Password

/auth/forgot-password

What this page is

A simple form where a user who forgot their password requests a reset link. The user enters their email address and submits. Supabase sends a password reset email with a magic link. Clicking that link takes the user to a page where they choose a new password.

After submission

The page shows a success message confirming the email was sent (regardless of whether the email exists in the system, for security). A link to return to the login page is provided.

/auth/verify-email

What it shows

A large mail icon, the heading "Verify Your Email", the email address the link was sent to (from the URL parameter), a reminder to check the spam/junk folder, and two buttons: "Log In" and "Back to Home".

This is purely informational

The actual verification happens via the magic link in the email — this page just tells the user to go check their inbox.

Complete Profile

/auth/complete-profile

Page 24

Pet Listings

/pets

What this page is

A dedicated section showing only listings in the "Pet" category — lost and found animals. It is separate from the general search page because pet listings have unique filterable fields (species, breed, colour, microchip number) that deserve their own UI.

Filters available

  • Keyword search across name and description.
  • Type filter: All / Lost / Found.
  • Species filter: All, Cat, Dog, Bird, Rabbit, Hamster, Other — each shown with an emoji.

What each listing card shows

  • Photo (or species emoji if no photo).
  • Lost/Found badge and species badge.
  • Breed, colour, and age chips.
  • Microchip number (if provided) — shown in blue so vets can spot it immediately.
  • Location, view count, and time since posting.

Tips section

Below the grid there is an informational box with advice on posting an effective pet listing — multiple photos, microchip details, distinctive markings, reward offers.

Who uses it
  • Pet owners who lost their animal
  • People who found a stray animal

Admin Panel

/admin

What this page is

The back-office control centre of the platform, accessible only to designated admin users. It is a single-page panel divided into tabs, each covering a different area of platform management.

Tab — Dashboard

Overview statistics: total listings (and new in the last 7 days), registered users, pending listings, open support sessions, resolved items, total sightings. A bar chart shows daily listing volume (lost vs. found) for the past 30 days.

Tab — Listings

A full table of all listings with search and filter. Actions per listing: Approve (makes it public), Flag (returns it to the owner for editing), Delete. Pending listings are highlighted in amber.

Tab — Users

A list of all registered users with email, join date, listing count, resolved count, and ban status. Actions: ban/unban a user, add their email to the permanent blacklist (prevents re-registration).

Tab — Reports

All abuse reports submitted by users (for listings or profiles), showing the reporter, reason, and status. The admin can mark reports as resolved and optionally act on the reported content.

Tab — Support

Live customer support management. Shows all open and closed support sessions. The admin can click into any session to see the full chat history and reply in real time. Sessions can be closed from this view.

Tab — Success Stories

Lists all submitted success stories with approval status. The admin can approve (makes it public) or delete each story.

Tab — User Requests

Contact form submissions from the /destek page appear here. The admin can read the full message, write a response, mark it as resolved, or delete it.

Tab — Sightings

All "I saw it" sighting reports, showing the item, reporter, location, and note. Useful for spotting patterns or verifying suspicious reports.

Access

Restricted to email addresses in the admin configuration. All other users are redirected to the home page.
